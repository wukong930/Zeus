# Zeus 产品需求文档 (PRD)

> 版本: 1.0 | 日期: 2026-04-30 | 状态: 已评审

## 1. 产品概述

### 1.1 产品定位

Zeus 是一个面向专业期货交易员和交易团队的智能分析平台。系统自动监控商品期货市场的多维信号，通过统计检验、对抗验证和自校准机制生成高质量预警，结合品种成本模型和持仓状态提供科学的交易建议。

### 1.2 目标用户

- 期货交易员（个人/团队）
- 产业客户（采购、销售决策者）
- 投研分析师

### 1.3 核心价值主张

- **更准确的预警**：对抗验证 + 自校准权重，减少假信号
- **产业级成本洞察**：从原料到出厂的完整成本链，一眼看懂生意赚不赚钱
- **持仓感知的建议**：交易建议与实际持仓科学结合，避免重复暴露
- **动态人机协作**：系统置信度决定人的介入程度，而非静态模式切换

## 2. 功能需求

### 2.1 数据采集与 ETL

**F-ETL-01**: 市场数据采集
- 支持 Tushare + AkShare + 爬虫三个数据源
- 覆盖国内四大交易所 50+ 品种 + 海外主要品种
- 交易日自动采集，频率可配置（默认：开盘/收盘各一次）
- 数据质量检查：缺失率 > 20% 标记降级

**F-ETL-02**: 产业数据采集
- 库存/仓单、现货价格、基差
- 持仓排名（龙虎榜）、波动率指数
- 宏观数据（PMI、汇率、BDI）
- CFTC 持仓报告（海外品种）

### 2.2 信号检测

**F-SIG-01**: 六类信号评估器
- 价差异常（Z-score 偏离）
- 基差变动（现货-期货基差异常）
- 动量信号（价格趋势突破）
- 市场状态转换（regime shift 检测）
- 库存冲击（库存水平异常变动）
- 事件驱动（新闻/政策事件触发）

**F-SIG-02**: DB 驱动的监控列表
- 监控品种对从数据库读取，支持增删改
- 支持自定义检测优先级（1-3 级）
- 支持品种级自定义阈值覆盖
- 持仓品种自动加入高优先级监控

**F-SIG-03**: 持仓感知阈值
- 持仓品种的检测阈值自动降低（× 0.8），提高灵敏度
- 持仓品种的上下游关联品种自动进入监控（传导图激活）

### 2.3 对抗验证

**F-ADV-01**: 零假设检验
- 每个信号触发前执行 Bootstrap 置换检验（1000 次迭代）
- 验证信号是否可与随机噪声区分（p-value < 0.05）

**F-ADV-02**: 历史组合检验
- 对当前信号组合计算哈希
- 查询历史上相同组合的命中率
- 命中率 < 30% 且样本量 > 20 时降级

**F-ADV-03**: 结构性反驳
- 遍历品种传导图寻找反向论据
- 检查季节性反转因素
- 检查替代品压力

**F-ADV-04**: 对抗结果处理
- 三项全部失败 → 信号抑制
- 部分失败 → 置信度惩罚（× 0.7）
- 全部通过 → 正常流转
- 所有结果记录到 `adversarial_results` 表，可追溯

### 2.4 评分与校准

**F-SCR-01**: 多维评分
- 优先级评分：Z-score 幅度 + 半衰期 + ADF 平稳性 + 置信度
- 组合适配评分：与现有持仓的冲突/互补程度
- 保证金效率评分：保证金占用比例

**F-SCR-02**: 校准循环
- 每个信号源独立追踪命中率（90 天滚动窗口）
- 按 (信号类型, 板块, 市场状态) 三元组分别统计
- 评分权重由贝叶斯更新自动调整，不再硬编码
- 信号衰减检测：命中率 < 0.3 持续 30 天标记衰减，权重 × 0.5

**F-SCR-03**: 人工反馈闭环
- 人工审批/拒绝/修改记录作为校准数据
- 人工决策与信号结果关联，用于评估人机协作效果

### 2.5 预警与分级

**F-ALT-01**: 混合决策路由
- 确定性路径（90%+）：规则分级 + 模板叙事，无 LLM 调用
- LLM 路径（10%）：信号矛盾、模糊置信度、无历史先例、跨板块传导时触发
- LLM 输出强制结构化 JSON，无效输出回退确定性路径

**F-ALT-02**: 四级预警分类
- L0 市场全景：宏观环境变化
- L1 板块评估：板块级别的方向判断
- L2 品种信号：具体品种的交易信号
- L3 交易建议：含入场/止损/止盈的完整建议

**F-ALT-03**: 置信度分层路由
- \>85% + 风控达标 → `auto`：自动执行，事后通知
- 60-85% → `notify`：自动执行，实时通知，人可在窗口期否决
- <60% → `confirm`：暂停，等待人工确认
- 信号冲突 → `arbitrate`：展示矛盾点，等待人工仲裁

**F-ALT-04**: 实时推送
- SSE 实时推送到前端
- 飞书 Webhook 通知
- 支持 Email / 自定义 Webhook（可扩展）

### 2.6 持仓管理

**F-POS-01**: 最小化持仓录入
- 核心字段：品种、方向（多/空）、手数、均价、开仓日期
- 快捷操作：平仓、加仓、减半（一键操作）
- 交易建议采纳时自动预填持仓记录

**F-POS-02**: 持仓驱动的系统行为
- 持仓品种自动升级监控等级（阈值 × 0.8）
- 传导图自动激活关联品种监控
- 组合风控实时计算（VaR、集中度、相关性）
- 新建议与持仓冲突时标记警告

**F-POS-03**: 数据腐烂防护
- 持仓数据超过 N 天未更新 → 主动提醒确认
- 长期不更新 → 降级为无持仓模式
- 不基于过期数据给出建议

**F-POS-04**: 动态止盈止损
- 基于成本模型 + 波动率 + 持仓均价计算建议止盈止损位
- 随市场变化持续更新

### 2.7 成本模型

**F-CST-01**: 品种成本链
- 完整链路：原料成本 → 加工成本 → 损耗/回收率 → 运输仓储 → 人工水电 → 税费 → 单位成本
- 输出：单位成本、毛利、净利、盈亏平衡价格
- 支持动态调价：修改任一输入参数，实时重算全链路

**F-CST-02**: 优先品种成本模型
- 橡胶（NR→RU）：天然胶采购 → 加工（乳胶/烟片/标胶分级）→ 运输 → 成品
- 黑色系产业链：
  - 焦煤（JM）→ 焦炭（J）：配煤比、炼焦成本、副产品回收
  - 铁矿石（I）+ 焦炭（J）→ 螺纹钢（RB）：高炉利润模型
  - 热卷（HC）：热轧加工费差异

**F-CST-03**: 成本快照与信号集成
- 每日自动重算成本快照，存入 `cost_snapshots` 表
- 利润率 < -5% 持续 2 周 → 触发"产能收缩预期"信号
- 利润率由负转正 → 触发"复产预期"信号
- 价格触及盈亏平衡线 → 触发"成本支撑/压力"信号

**F-CST-04**: 前端成本展示
- 品种成本分解瀑布图（原料 → 加工 → 物流 → 税费 → 总成本）
- 利润率趋势图（历史利润率 + 当前位置）
- 盈亏平衡线标注在价格图上
- 动态调价计算器（用户可修改参数，实时看结果）

### 2.8 板块智能

**F-SEC-01**: 板块级模型
- 黑色系：高炉利润模型（螺纹钢价 - 1.6×铁矿 - 0.5×焦炭 - 加工费）
- 能化：裂解价差、炼厂利润（后续阶段）
- 农产品：压榨利润、种植周期（后续阶段）
- 每个板块模型输出 conviction_float（-1 到 +1 的方向确信度）

**F-SEC-02**: 品种传导图
- 品种间关系：上下游、替代品、库存传导、国内外联动、成本驱动
- 传导元数据：影响权重、滞后天数、方向
- 持仓品种自动激活关联节点

### 2.9 场景推演

**F-SCN-01**: 触发机制
- 手动启动：用户输入假设条件
- 条件触发：Alert Agent 在置信度模糊时请求推演
- 独立于主预警链路，不阻塞常规流程

**F-SCN-02**: 推演引擎
- Monte Carlo 价格路径模拟（基于传导图）
- What-if 假设检验（用户定义场景）
- LLM 生成可读的情景描述报告

**F-SCN-03**: 人机协作
- 推演结果需人工审核反馈
- 支持迭代式推演（调整假设 → 重新推演）

### 2.10 风控

**F-RSK-01**: 组合风险
- VaR / CVaR（95% 和 99% 置信度）
- 相关性矩阵
- 板块集中度监控

**F-RSK-02**: 压力测试
- 历史极端日提取
- 自定义压力场景
- 组合在压力下的预期损失

**F-RSK-03**: 风控联动
- 保证金使用率超限 → 新建议降级为 watchlist_only
- 板块集中度超限 → 同板块新建议拦截
- 持仓品种出现反向信号 → 优先推送

### 2.11 回测

**F-BKT-01**: 策略回测
- vectorbt 引擎：均值回归、动量突破、通道突破、事件驱动
- 指标：Sharpe、最大回撤、胜率、Calmar、Sortino、Omega
- 成本模型：手续费、滑点、保证金成本

**F-BKT-02**: 因果验证
- DoWhy 因果推断框架
- 安慰剂检验（50 次模拟）
- 输出：因果效应、p-value、置信区间

**F-BKT-03**: 滚动前推验证
- Walk-forward analysis
- 样本外稳定性检验

## 3. 非功能需求

### 3.1 性能
- 信号检测全流程（6 评估器并行）< 30 秒
- 前端页面首屏加载 < 2 秒
- SSE 预警推送延迟 < 5 秒
- 成本模型重算 < 3 秒

### 3.2 可靠性
- 事件总线死信队列，失败事件可重试
- 调度器健康追踪，连续失败 ≥ 3 次标记降级
- 数据源故障时自动降级到备用源
- 数据库连接池（max 20, idle_timeout 30s）

### 3.3 安全
- API 认证：Bearer Token
- LLM API Key 加密存储（AES-256）
- 环境变量管理敏感配置
- 非 root 用户运行容器

### 3.4 可观测性
- 事件审计日志（`event_log` 表）
- 信号全链路追踪（correlation_id）
- 调度器执行日志
- 校准指标仪表盘

## 5. 用户界面

| 页面 | 功能 | 优先级 |
|------|------|--------|
| Dashboard | 市场热力图、预警流、风控概览、账户状态 | P0 |
| Alerts | 实时预警列表、筛选、详情（含对抗验证结果） | P0 |
| Positions | 持仓录入/管理、组合风控、传导图激活状态 | P0 |
| Cost Models | 品种成本分解、利润趋势、盈亏平衡、动态调价 | P0 |

| Backtest | 回测运行器、策略对比 | P2 |
| Analytics | 组合分析、信号校准仪表盘 | P2 |

### 5.2 关键交互

- 预警实时推送（SSE），新预警时页面顶部提示
- 持仓一键录入（5 字段），交易建议采纳时自动预填
- 成本模型动态调价计算器（滑块/输入框修改参数，实时重算）
- 人工仲裁界面：展示矛盾信号 + 对抗验证结果，支持审批/拒绝/修改
